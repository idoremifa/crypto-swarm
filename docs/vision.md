# Vision — 장기 방향

이 문서는 **지금 만드는 것**이 아니라 **언젠가 만들 수 있는 것**을 적어둔다. MVP 구현 중에 "이건 나중에"라고 미룬 아이디어의 저장소 역할이다. 실제 구현 계획은 [mvp.md](./mvp.md)에 있다.

## 가능한 장기 모습

여러 전략 에이전트가 시장 국면에 맞춰 서로 다른 관점으로 신호를 만든다.
운영자는 CLI나 대시보드로 승인·kill switch를 관리한다.
텔레그램은 알림과 리포트만 보낸다.
학습 루프와 전략 연구 기능은 과거 거래 결과를 바탕으로 개선 후보를 제안할 수 있지만, 자동으로 운영 설정을 바꾸지는 않는다.

## 1차 범위 (MVP)

- 업비트 현물 KRW 마켓
- BTC, ETH 등 유동성 높은 소수 페어
- Long/flat (매수하거나 현금으로 쉰다)
- 지정가/시장가 주문
- backtest(과거 데이터 검증) → paper(실제 주문 없는 모의 운영) → live-small(소액 실거래) 순차 전환

## 범위 밖 (영구 또는 먼 미래)

구조적으로 하지 않기로 한 것의 규칙 출처는 [AGENTS.md §1.4 금지된 기능](../AGENTS.md#14-금지된-기능)과 [§1.1 주문과 자금](../AGENTS.md#11-주문과-자금)에 있다. 이 문서는 해당 규칙을 반복하지 않는다.

이 프로젝트의 핵심 위험(실계좌 안전성·감사 가능성·단일 거래소)과 무관한 영역은 방향에서도 제외한다.

- GPU 추론 최적화, KV cache 압축, vLLM/Triton 서빙

## Post-MVP 백로그

MVP가 live-small에서 안정적으로 돌아간 뒤에 검토할 순서다.

### 다음 순위 (MVP 직후 고려)

- **전략 에이전트 확장**: Volatility Risk Agent처럼 정량화하기 쉬운 보조 신호부터 추가한다.
- **Post-Trade Review Service**: 거래 단위로 "왜 좋았고 왜 나빴는지" 기록한다.
- **Agent Score Service**: 에이전트별 성과를 점수화한다.
- **운영 리포트 개선**: 운영자가 매일 보는 손익, 수수료, 장애, 승인 이력을 더 읽기 쉽게 정리.

### 중간 순위 (성과가 쌓인 뒤)

- **Learning Loop**: 거래 결과에서 개선 후보(`LearningProposal`)를 생성. 자동 적용은 하지 않고 Approval Gate 승인을 거친다.
- **Strategy Lab**: 전략 후보 자동 생성 → backtest → walk-forward 검증. walk-forward는 기간을 순서대로 밀어가며 반복 검증하는 방식이다. 운영 반영은 Approval Gate를 통한다.
- **다른 거래소 어댑터**: 바이낸스, 바이비트 현물. 업비트 live-small 운영이 안정된 뒤 검토한다.
- **Fee Rate Monitor**: 업비트 공식 페이지 변경을 자동 감지한다. MVP에서는 수동으로 점검한다.
- **Prometheus + Grafana + Sentry**: 본격적인 관측에 사용한다.
- **운영 대시보드(웹 UI)**: Approval Gate 승인, 실험 리포트, 에이전트 trace 조회.

### 먼 순위 (연구 단계)

> MVP에도 `regime`이라는 단어가 Feature Service 출력(`trend/range/high-vol/low-vol` 라벨)과 Regime Filter Agent에 등장한다. 아래에서 말하는 것은 **LLM 기반 Regime Research Agent**이며 서로 다른 개념이다.

- **토론형 에이전트 구성**: Regime Research / Bull Research / Bear Research / Debate Agent. TradingAgents 프로젝트를 참고한다. 실용 가치를 검증해야 한다.
- **뉴스/공시/소셜 요약 에이전트**: LLM 기반이다. 출력은 `ContextSignal`로만 반영한다.
- **장기 메모리 + 세션 검색**: 긴 실험 로그와 운영 메모를 검색한다.
- **플랫폼 교체 검토**: 자체 asyncio 오케스트레이터나 feature 저장 방식이 유지보수 한계에 닿을 때만 외부 도구를 검토.

### 하지 않기로 결정한 것

보안·권한 경계 때문에 구조적으로 하지 않기로 한 것은 [AGENTS.md §1.2 권한과 비밀](../AGENTS.md#12-권한과-비밀)과 [§1.3 자동 변경](../AGENTS.md#13-자동-변경)에 있다. 규칙 본문은 AGENTS.md에서만 관리한다.

## 참고한 외부 프로젝트

아래 프로젝트에서 일부 아이디어를 가져왔지만, **그대로 복사하지 않는다.** 우리 범위(업비트 현물, 개인 운영, 실계좌 안전성)에 맞게 선택적으로 반영한다.

| 프로젝트 | 가져올 것 | 그대로 쓰지 않을 것 |
|---|---|---|
| TradingAgents | 역할 분리, 상승/방어 관점 토론 | LLM이 주문 판단을 끝까지 맡는 구조, 주식 중심 데이터 모델 |
| auto-researchtrading | `uv` 기반 반복 실험, 단일 전략 인터페이스, 벤치마크, 점수식, 결과 TSV | 선물 중심 전략, 공개 성과 수치 기대 |
| turboquant | 모듈형 검증, 성능 벤치마크, 과장 주장에 대한 audit 태도 | 추론 최적화 중심 구조, GPL 의존 |
| Vibe-Trading | CLI, 실행 trace, 세션 검색, 백테스트 검증, 리포트 | 너무 많은 시장/도구/스킬을 첫 버전에 넣는 것 |

## 이 문서를 갱신하는 시점

- 새로운 아이디어가 나와 MVP에 넣기는 아깝고 잊기는 아쉬울 때 갱신한다.
- 어떤 기능을 **영구히 하지 않기로** 결정했을 때 갱신한다. AGENTS.md 규칙으로 올릴지 먼저 판단한다.
- MVP 후속 Phase 방향이 바뀔 때 갱신한다.

이 문서에 넣었다고 해서 만들기로 약속한 것은 아니다. **제안 + 고민 흔적**이다.
